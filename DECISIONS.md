# Decisions

Standing decisions for co-cog / Bramble B1. Format: `date — decision — rationale`.
Grown by promotion of blessed ledger assumptions at review close.

- 2026-07-05 — CaptureService refuses a half-permissioned start via a plain `startService()`: it records the refusal (MetricsSink event + a test-visible flag) and `stopSelf()`s *before* any `startForeground()` or microphone access. Callers must not launch it via `startForegroundService()` while half-permissioned — a microphone-typed `startForeground()` itself requires `RECORD_AUDIO`, so no valid foreground call exists then; hardening that path is T2's when it wires the real start. — Establishes the D18 refusal contract the rest of the capture service builds on (promoted from T1 ledger L5).
- 2026-07-05 — CaptureService actively records (calls `AudioRecord.startRecording()`), not merely constructs, while holding the microphone; it never reads from the stream. Fail-closed on any init/record failure: release + `stopSelf()`. — A merely-constructed `AudioRecord` does not acquire the input device or light the OS mic indicator, so future tasks that observe or gate on "is the mic held" (T3 state machine, T11 mute, T14 contention suspend) must treat an actively-recording-but-undrained stream as the mic-held state, not construction alone (promoted from T2 ledger L3).
- 2026-07-06 — The capture state machine (`com.cocog.capture.state`: `CaptureState`, `CaptureEvent`, `StateMachine.transition()`) is the standing contract for states CAPTURING/MUTED/SUSPENDED/THROTTLED/STOPPED and events Start/Stop/Mute/Unmute/MicContentionDetected/MicContentionResolved/ThermalSevere/ThermalRecovered. MUTED is a hard sink: any event other than Unmute or Stop while MUTED returns `IllegalTransition`, never a silent no-op. — Binds T4 (notification), T11 (mute), T14 (contention), T15 (thermal) to this exact vocabulary and rejection semantics; do not invent parallel event types or treat unexpected events while muted as ignorable (promoted from T3 ledger L1, L2).
- 2026-07-06 — Mute-from-STOPPED is `IllegalTransition`, not an exception folded into the MUTED-beats-everything property — a stopped/crashed service holds no microphone, so there is nothing for mute to override. — T4 and T11 must not assume mute is reachable from every state without exception; STOPPED sits outside the mute-wins property by design (promoted from T3 ledger L6).
- 2026-07-06 — `MUTED → CAPTURING` via `Unmute` does not re-evaluate thermal or contention conditions — the pure state machine has no memory of what was true before the mute. — Whichever task wires real unmute (T11) or contention/thermal detection (T14, T15) must have its Android-side caller re-assert current thermal/contention state itself before or after driving `Unmute`, rather than trusting the bare transition to CAPTURING as proof those conditions have cleared (promoted from T3 ledger L8).
